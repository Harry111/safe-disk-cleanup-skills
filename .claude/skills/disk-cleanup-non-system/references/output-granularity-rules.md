# 输出粒度规则

默认输出目录级、类别级、汇总级结果，不输出海量单文件清单。

## 推荐输出

- 大目录；
- 大文件夹；
- 缓存类别；
- 应用级占用；
- Profile 级浏览器缓存；
- 项目级依赖；
- 保守/标准/激进可清理潜力；
- 需要用户确认的路径。

## 不推荐输出

- 浏览器 Cache 内每个文件；
- node_modules 内每个包文件；
- .git objects；
- 日志目录里每个小日志；
- 临时目录里每个小文件。

## 建议格式

专项分析可以类似：

```text
浏览器缓存：约 6.2GB
旧 Edge 数据：约 618MB
微信纯缓存：约 1.7GB
pnpm/npm/node_modules：约 2GB
旧版本软件：约 2GB
日志：约 570MB
临时文件：约 420MB
保守合计：约 8-10GB
激进合计：约 14GB
```

执行前必须输出确认清单表格。
